# Drive Storage Card 0.3.2

Hotfix release addressing post-release bot feedback on dangerous-action editor
state handling.

## Highlights

- Fixed the editor dangerous-entity toggle flow so enabling one dangerous row
  (for example `shutdown`) while `show_dangerous_actions` is disabled does not
  implicitly enable other dangerous rows.
- Preserved user intent by keeping non-selected dangerous keys hidden in
  `hide_entities` during the auto-enable transition.
- Added regression coverage for this exact editor flow.

## Validation

- TypeScript typecheck, ESLint, Vitest and Vite build.
- HACS/custom-card compatibility checks.
- License and third-party notice checks.
- Local anonymization scan and GitHub-surface anonymization scan.
- Live Home Assistant deploy smoke and live entity audit.
- Release metadata and workflow checks.
